# HomePick

> AI 기반 부동산 시장 분석을 바탕으로, 사용자의 주거 조건에 맞는 매물을 탐색하고 비교할 수 있도록 설계한 프론트엔드 프로젝트입니다.

## 프로젝트 소개

HomePick은 복잡하게 흩어진 부동산 정보를 사용자의 관점에서 이해하기 쉽게 정리하고, 시장 데이터와 선호 조건을 결합한 맞춤형 매물 탐색 경험을 제안합니다. 직관적인 검색 흐름, 정보 밀도가 높은 카드 UI, 반응형 레이아웃을 중심으로 부동산 서비스의 핵심 화면을 구현합니다.

> 현재 저장소는 화면 설계와 정적 프론트엔드 구현을 중심으로 구성되어 있습니다. AI 추천, 실거래 데이터 연동, 인증 및 관리자 기능은 향후 백엔드 연동 단계에서 구현할 예정입니다.

## 프로젝트 목표

- 사용자의 지역, 예산, 주거 형태, 선호 조건을 반영하는 매물 탐색 경험 설계
- 시장 분석과 추천 정보를 한 흐름 안에서 제공하는 정보 구조 구축
- 데스크톱과 모바일 환경에서 일관되게 사용할 수 있는 반응형 UI 구현
- 공통 컴포넌트와 스타일 가이드를 활용한 유지보수 가능한 프론트엔드 기반 마련

## 기획 의도

부동산 탐색은 정보의 양이 많고 비교 기준이 복잡해 의사결정에 시간이 오래 걸립니다. HomePick은 단순한 매물 나열을 넘어, 사용자가 자신의 상황에 맞는 조건을 빠르게 정리하고 시장 맥락 속에서 매물을 판단할 수 있도록 돕는 것을 목표로 합니다. 이를 위해 검색, 시장 분석, 관심 매물 관리, 운영 관리의 핵심 여정을 하나의 서비스 경험으로 연결했습니다.

| 담당 | 화면 / 파일 | 핵심 기능 | 담당 업무 |
| --- | --- | --- | --- |
| 김길동 | 메인 페이지 / `index.html` | 서비스 소개, 히어로 영역, 주요 탐색 진입점, 추천 매물 카드 | 전체 정보 구조를 설계하고 메인 레이아웃·공통 진입 UI·반응형 그리드의 구현 방향을 수립했습니다. |
| 홍길동 | 매물 검색 / `templates/sub/product.html` | 조건별 필터, 검색 결과 목록, 카드형 매물 정보, 상세 화면 진입 | 검색 화면의 사용자 흐름과 필터 인터랙션을 설계하고, 결과 리스트의 카드 컴포넌트 및 상태별 UI를 구현했습니다. |
| 박미금 | 마이 홈픽 / `templates/sub/myhomepick.html`* | 관심 매물, 최근 본 매물, 알림 및 개인 설정 | 개인화 영역의 IA를 설계하고, 관심 매물 관리·최근 조회 이력·설정 화면의 UI 시나리오를 작성했습니다. |
| 김서연 | 관리자 페이지 / `templates/sub/dashboard.html`* | 매물·회원 관리, 운영 현황, 시장 지표 대시보드 | 관리자 화면의 대시보드 구조와 데이터 테이블·통계 카드·운영 관리 플로우를 설계했습니다. |

\* 기획 단계의 화면입니다. 현재 저장소에 포함된 구현 파일은 `index.html`, `templates/common/header.html`, `templates/sub/product.html` 등이며, 나머지 화면은 확장 계획에 따라 추가됩니다.

```text

├── static/
│   └── css/

│       └── main.css               # 메인 페이지 스타일
├── templates/
│   ├── common/
│   │   ├── header.html            # 공통 헤더 템플릿
│   │   └── footer.html            # 공통 푸터 템플릿
│   └── sub/
│       └── product.html           # 매물 목록 화면
└── prompt.txt                     # 프로젝트 요구사항
```

## UI 핵심 설계

아래 기능은 서비스 완성 단계에서 적용할 UI/UX 설계 기준입니다.

- **반응형 레이아웃**: Flexbox와 Bootstrap Grid를 기반으로 모바일부터 데스크톱까지 자연스럽게 확장되는 화면 구성
- **계층형 내비게이션**: 메인 메뉴와 하위 메뉴를 명확히 구분해 매물 탐색, 시장 분석, 개인화 메뉴로 빠르게 이동
- **카드 기반 정보 구조**: 가격, 위치, 면적, 주요 옵션을 한눈에 비교할 수 있도록 핵심 정보를 카드 단위로 구성
- **조건 중심 검색 경험**: 지역·가격·주거 형태·면적 등 자주 사용하는 조건을 우선 배치하고 결과 수와 함께 즉시 피드백 제공
- **일관된 디자인 토큰**: 색상, 타이포그래피, 간격을 CSS 변수로 관리하여 화면 간 시각적 일관성 유지
- **접근성 고려**: 시맨틱 태그, 명확한 링크 텍스트, 키보드 탐색 및 ARIA 속성 적용을 확장 기준으로 설정

## 개발 일정

5일간, 하루 2시간을 기준으로 진행한 프론트엔드 개발 일정입니다.

| 일차 | 목표 | 주요 작업 |
| --- | --- | --- |
| Day 1 | 프로젝트 기반 구축 | 요구사항 정리, 화면 구조 설계, Git 저장소 구성, 공통 스타일 가이드 정의 |
| Day 2 | 공통 UI 및 메인 구현 | 헤더·내비게이션 구현, 메인 레이아웃 구성, 핵심 진입 영역과 반응형 기준 적용 |
| Day 3 | 매물 검색 화면 구현 | 검색 조건 UI, 결과 카드 구조, 목록 화면의 사용자 흐름 설계 |
| Day 4 | 개인화·관리 화면 설계 | 마이 홈픽 및 관리자 대시보드의 와이어프레임, 컴포넌트와 상태 정의 |
| Day 5 | 검수 및 배포 | 화면 간 UI 점검, 반응형 검수, 코드 정리, GitHub Pages 배포 |

## 향후 개발 예정

프론트엔드 확장 범위에서 다음 기능을 단계적으로 구현할 예정입니다.

- 매물 검색 필터와 정렬·페이지네이션 인터랙션
- 지도 기반 탐색 UI 및 지역별 매물 시각화
- 관심 매물, 최근 본 매물, 비교함을 포함한 마이 홈픽 화면
- 시장 지표·지역 트렌드를 보여 주는 대시보드와 차트 UI
- 다국어, 다크 모드, PWA를 고려한 사용자 경험 고도화

## 개선 및 보완 계획

- 이미지 최적화, 지연 로딩, 번들 분리를 통한 초기 로딩 성능 개선
- 키보드 탐색, 명도 대비, 스크린 리더 안내를 포함한 웹 접근성 강화
- 재사용 가능한 컴포넌트 규칙과 상태 설계 문서화
- 빈 결과, 로딩, 오류 상태를 포함한 예외 시나리오 UI 보완
- 실제 데이터 연동 전 목업 데이터의 구조와 표시 규칙 표준화

## 기대 효과

- 사용자는 복잡한 조건을 빠르게 좁히고, 비교 가능한 형태의 매물 정보를 확인할 수 있습니다.
- 서비스 운영자는 검색·관심·조회 흐름을 바탕으로 개인화 기능을 확장할 기반을 확보할 수 있습니다.
- 팀은 공통 스타일과 화면 구조를 바탕으로 신규 페이지와 기능을 일관되게 추가할 수 있습니다.
